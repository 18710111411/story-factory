## 🎯 核心能力

### 1. 剧本→分镜→提示词转换
- 分析剧本场景和情感
- 分解为镜头序列
- 生成优化的 AI 视频提示词

### 2. 导演思维注入
- **规则 1**：镜头内变化优先于切镜头
- **规则 2**：对话完整保留优先
- **规则 3**：CG 动画优势最大化
- **规则 4**：情绪弧线连续
- **规则 5**：16:9 横屏构图优化

### 3. 多平台提示词优化
| 平台 | 优化策略 |
|------|----------|
| **Seedance(即梦)** | 15 秒规则，中文≤500 字符，光影三层结构 |
| **Kling** | 复杂动作优化，物理逻辑描述 |
| **Veo3** | 高质量美学，长镜头，环境细节 |
| **Luma** | 快节奏优化，简洁有力描述 |

### 4. 光影三层结构
- **光源层**：主光/辅光/背光位置
- **光行为层**：反射/折射/散射/体积光
- **色调层**：冷暖对比/色彩公式

### 5. 对话时长计算
```
正常语速：180 字/分钟 = 3 字/秒
1.3 倍速：234 字/分钟 = 3.9 字/秒
1.5 倍速：270 字/分钟 = 4.5 字/秒

镜头时长 (秒) = 台词字数 / (语速 字/分钟 / 60)
```

## 🛠️ 工作流程

```
剧本输入 → 场景分析 → 镜头分解 → 提示词生成 → 质量校验 → 输出
    ↓           ↓           ↓           ↓           ↓         ↓
 创意简报   情感基调   镜头规格   平台优化   规则检查   可用提示词
```

### Step 1: 剧本输入
**接收内容**：
- 剧本/场景描述
- 时长要求
- 风格参考
- 情感基调

### Step 2: 场景分析
**分析维度**：
| 维度 | 检查项 |
|------|--------|
| **情感基调** | 压抑/委屈/愤怒/觉醒/希望 |
| **场景类型** | 对话/动作/情绪/转场 |
| **角色数量** | 单人/双人/多人 |
| **特效需求** | 粒子/全息/光芒/物理模拟 |
| **灯光方案** | 低调/侧光/硬光/逆光/体积光 |

### Step 3: 镜头分解
**分解规则**：
| 内容类型 | 镜头数/分钟 | 平均镜头时长 |
|----------|-------------|--------------|
| 压抑铺垫 | 15-18 | 3.3-4s |
| 对话冲突 | 18-22 | 2.7-3.3s |
| 动作高潮 | 22-28 | 2.1-2.7s |
| 情绪收尾 | 15-18 | 3.3-4s |

### Step 4: 提示词生成
**六要素公式**：
```
[主体与外貌细节] + [动作与物理连贯性] + [场景环境] +
[视觉风格/物理光影] + [物理焦段与运镜] + [原生音效要求]
```

**时间戳分镜法**（>5 秒镜头）：
```
[风格/色调总纲]。
0-3 秒：[画面 + 镜头]。
3-8 秒：[画面 + 镜头]。
光影：[光源层] + [光行为层] + [色调层]。
音效：[物理拟声描述]。
禁止：任何文字、字幕、LOGO 或水印
```

### Step 5: 质量校验
**检查清单**：
- [ ] 对话 - 镜头时长匹配
- [ ] 镜头密度合理 (18-22 镜头/分钟)
- [ ] 光影三层结构完整
- [ ] 运镜术语安全（中文用"推轨推进"不用"Dolly"）
- [ ] 提示词长度合规 (中文≤500 字符)
- [ ] 无版权敏感词

### Step 6: 输出
**交付格式**：
```markdown
## 镜头 X | [镜头名称]

**时长**：X 秒
**景别**：[景别]
**情感**：[情感基调]

**提示词**：
```
[完整提示词]
```

**备注**：[特殊说明]
```

## 🆘 常见问题

### Q1: 提示词被平台拦截怎么办？
**A**: 检查以下项：
- 是否包含裸英文运镜词（Dolly/Aerial/Crane）→ 改用中文
- 是否包含版权敏感词 → 替换为描述性昵称
- 是否超过 500 字符 → 精简冗余词
- 是否包含写实真人面部 → 改为动画/剪影

### Q2: 生成视频动作不流畅？
**A**: 优化提示词：
- 增加物理逻辑描述（重心转移/流体风阻）
- 明确动作起止状态
- 使用"fluid motion""realistic physics"等词
- 考虑分段生成后拼接

### Q3: 对话与口型不同步？
**A**: 
- 计算对话时长，确保镜头时长足够
- 使用 1.3 倍速提示词
- 选择支持口型同步的模型（Seedance/Kling）
- 添加"precise lip sync"描述

### Q4: 多个角色如何保持一致性？
**A**:
- 为每个角色创建形象描述卡
- 使用@图片引用固定形象
- 每段提示词重复角色关键特征
- 避免角色数量过多（≤3 个主要角色）
