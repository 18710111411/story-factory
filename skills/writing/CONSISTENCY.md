> **目的**：建立系统性规则，避免 AI 写作中的逻辑错误、设定遗忘和上下文断裂问题

## B. 上下文管理流程

### B1. 写作前的设定回顾清单

```markdown
## 写作前检查清单 □

### 基础设定回顾（5 分钟）
- [ ] 阅读当前章节涉及的人物档案
- [ ] 回顾相关场景描述
- [ ] 确认时间线位置（第 X 天/第 X 年）
- [ ] 检查前章结尾的悬念/待解决问题

### 情节一致性检查
- [ ] 确认本章在主线中的位置
- [ ] 回顾需要回收的伏笔
- [ ] 确认人物当前状态（情绪/伤势/物品）
- [ ] 检查是否有未解释的情节跳跃

### 设定约束确认
- [ ] 世界观规则是否允许本章事件
- [ ] 人物能力是否在合理范围内
- [ ] 场景转换是否有逻辑支撑
- [ ] 时间流逝是否合理

### 上下文衔接
- [ ] 与前章的过渡是否自然
- [ ] 是否需要简要回顾关键信息
- [ ] 是否有突兀的新设定引入
- [ ] 语气/风格是否一致

### 准备注入的上下文
```
关键信息摘要（200 字内）：
[粘贴此处]
```

---
**检查完成时间**：[时间]
**检查者**：[AI/人类]
```

### B2. 写作中的实时检查点

```markdown
## 实时检查点（每写 500-1000 字触发）

### 检查点触发条件
- 场景转换时
- 新人物登场时
- 时间跳跃时
- 引入新设定时
- 人物做出重大决定时

### 快速自检问题
1. **人物一致性**：这个行为符合人物性格吗？
2. **逻辑合理性**：这件事为什么会发生？有足够铺垫吗？
3. **设定约束**：这违反世界观规则吗？
4. **信息连贯**：读者能理解当前情况吗？
5. **节奏把控**：这里需要放慢/加快吗？

### 标记系统
在写作中使用标记标注需要后续处理的内容：
- `[CHECK: 时间线]` - 需要验证时间逻辑
- `[FIXME: 伏笔]` - 需要埋下/回收伏笔
- `[TODO: 设定]` - 需要补充设定说明
- `[WARNING: 矛盾]` - 可能存在前后矛盾
- `[NOTE: 回顾]` - 后续章节需要呼应

### 上下文注入格式
```
[CONTEXT]
当前场景：[场景名称]
在场人物：[人物列表]
时间位置：[具体时间]
前情提要：[1-2 句]
[END CONTEXT]
```

---
**最后检查点**：[字数位置]
**待处理标记**：[数量]
```

### B3. 写作后的一致性审查流程

```markdown
## 一致性审查流程（每章完成后执行）

### 第一阶段：自动审查（AI 执行）
**时间**：完成后立即
**内容**：
1. 扫描所有 `[CHECK]`、`[FIXME]`、`[TODO]` 标记
2. 对比人物档案，检查行为一致性
3. 验证时间线逻辑
4. 检查场景描述冲突
5. 识别可能的因果断裂

**输出**：审查报告（问题列表）

### 第二阶段：人工审查（人类执行）
**时间**：AI 审查后
**内容**：
1. 审阅 AI 提出的问题
2. 检查情感弧线是否连贯
3. 评估节奏是否合适
4. 确认主题表达是否清晰
5. 标记需要修改的段落

**输出**：修改建议

### 第三阶段：修正与验证
**流程**：
```
发现问题 → 分类优先级 → 逐一修正 → 再次验证
    │           │           │          │
 [记录]    [高/中/低]   [修改]    [通过？]
```

### 审查清单
- [ ] 人物性格无矛盾
- [ ] 时间线清晰合理
- [ ] 场景转换流畅
- [ ] 伏笔有交代
- [ ] 设定无冲突
- [ ] 信息不丢失
- [ ] 风格一致

### B4. 长文本分段管理策略

```markdown
## 长文本分段管理

### 分段原则
**按结构分**：
- 每章：3000-5000 字
- 每节：800-1500 字
- 每场景：500-1000 字

**按内容分**：
- 场景转换必分段
- 视角转换必分段
- 时间跳跃必分段
- 情绪转折可分段

### 上下文窗口管理
```
[完整作品]
    │
    ├─→ [卷/部]（5-10 万字）
    │       │
    │       └─→ [章]（3-5 千字）← 当前写作单元
    │               │
    │               ├─→ [节]（1 千字）
    │               └─→ [场景]（500 字）
    │
    └─→ [摘要层]
            ├─→ 作品摘要（500 字）
            ├─→ 卷摘要（200 字）
            └─→ 章摘要（50 字）
```

### 摘要层级系统
**作品级摘要**（500 字）：
- 核心主线
- 主要人物
- 世界观要点
- 主题思想

**卷/部级摘要**（200 字）：
- 本卷主线
- 涉及人物
- 关键事件
- 起止状态

**章级摘要**（50 字）：
- 本章事件
- 场景位置
- 人物状态变化

### 上下文注入策略
**写作时注入**：
```
[CONTEXT: 当前章节]

```

**跨章引用时**：
```

关联：[与当前的关系]
[END RECALL]
```

## C. 逻辑错误检测清单

### C1. 时间线一致性检查

```markdown
## 时间线检查清单 □

### 绝对时间验证
- [ ] 所有日期/时间是否明确
- [ ] 时间顺序是否合理（无倒流）
- [ ] 时间跨度是否符合事件复杂度
- [ ] 季节/天气与时间是否匹配

### 相对时间验证
- [ ] "三天后"、"翌日"等表述是否准确
- [ ] 人物年龄变化是否合理
- [ ] 事件间隔是否足够完成所述行为
- [ ] 同时发生的事件是否冲突

### 时间线绘制
```
第 1 天 ─→ 第 2 天 ─→ 第 3 天 ─→ 第 4 天
  │         │         │         │
[事件 A]  [事件 B]  [事件 C]  [事件 D]
  │         │         │         │
[章 1]    [章 2]    [章 3]    [章 4]
```

### 常见错误检查
- [ ] 人物是否同时出现在两地
- [ ] 事件是否在不可能的时间内完成
- [ ] 伤口愈合/物体变化时间是否合理
- [ ] 信息传递速度是否符合设定

### 检查工具
**时间线表格**：
| 章节 | 日期 | 时间 | 事件 | 涉及人物 | 备注 |
|------|------|------|------|----------|------|
|      |      |      |      |          |      |

**时间冲突检测**：
```
如果 [事件 A] 发生在 [时间 X]
且 [事件 B] 需要 [时长 Y]
那么 [事件 B] 最早完成时间 = [时间 X + 时长 Y]
验证：实际完成时间 ≥ 计算完成时间？
```

---
**检查者**：
**日期**：
**问题数**：

### C2. 因果关系验证

```markdown
## 因果关系检查清单 □

### 每个重大事件的因果链
```
[原因 1] ─┐
[原因 2] ─┼─→ [事件] ─→ [结果 1]
[原因 3] ─┘            └─→ [结果 2]
```

### 原因充分性检查
- [ ] 事件发生的原因是否充分
- [ ] 是否有足够的铺垫和伏笔
- [ ] 是否存在"天降神兵"式解决
- [ ] 人物动机是否清晰合理

### 结果合理性检查
- [ ] 结果是否与原因相称
- [ ] 是否有未解释的连锁反应
- [ ] 结果是否符合世界观规则
- [ ] 是否有人物行为无后果

### 因果断裂检测
**警示信号**：
- [ ] "突然"、"意外"频繁出现
- [ ] 人物决定缺乏心理描写
- [ ] 问题解决过于顺利
- [ ] 关键信息无来源

### 因果图示例
```
[主角童年创伤] ─→ [不信任他人]
                       │
                       ↓
              [拒绝帮助] ─→ [陷入困境]
                       │
                       ↓
              [被迫合作] ─→ [关系转变]
```

### 修正策略
1. **补充原因**：在前期添加铺垫
2. **调整结果**：使结果与原因匹配
3. **增加过渡**：补充中间环节
4. **删除事件**：如因果无法建立

---
**检查章节**：
**发现断裂**：
**修正方案**：
```

```markdown
## AI 写作助手 - 系统提示词

### 角色定义
你是一位专业的写作助手，专注于维护故事的一致性。你的核心职责是：
1. 追踪并维护所有设定（人物、场景、情节、世界观）
2. 检测并预警逻辑错误、设定冲突、前后矛盾
3. 协助作者保持叙事连贯性和风格统一
4. 提供建设性的修改建议

### 核心原则
- **准确性优先**：不确定的信息要标注，不编造设定
- **主动预警**：发现潜在问题立即提示
- **建设性反馈**：指出问题的同时提供解决方案
- **尊重原作**：修改建议要符合作者意图和作品风格

### 工作流程
```
接收输入 → 上下文匹配 → 一致性检查 → 输出建议
    │           │           │          │
 [文本]    [档案检索]   [多维度]   [结构化]
```

### 检查维度
每次写作输出前，你必须检查：
1. **人物一致性**：行为、语言、反应是否符合设定
2. **时间逻辑**：时间线是否连贯，时长是否合理
3. **空间逻辑**：场景转换是否清晰，位置是否合理
4. **因果逻辑**：事件是否有足够原因，结果是否合理
5. **设定约束**：是否违反世界观规则
6. **信息连贯**：是否有信息丢失或矛盾
7. **风格统一**：语气、用词、节奏是否一致

### 输出格式
```markdown
## 一致性检查报告

### ✅ 通过项
- [项目 1]
- [项目 2]

### ⚠️ 需要注意
- [问题 1]：[说明] + [建议]
- [问题 2]：[说明] + [建议]

### ❌ 需要修正
- [问题 1]：[说明] + [建议修改方案]

### 📝 补充建议
- [优化建议 1]
- [优化建议 2]
```

### 特殊指令
- 当用户输入 `[CHECK]` 时，执行完整一致性检查
- 当用户输入 `[RECALL]` 时，检索相关上下文
- 当用户输入 `[UPDATE]` 时，更新设定档案
- 当用户输入 `[SUMMARY]` 时，生成当前摘要

### 限制条件
- 不主动改变核心设定
- 不提供与作品风格冲突的建议
- 不替代作者做创造性决策
- 对不确定的信息标注 `[需确认]`

### 日常写作流程
```
1. 写作前（10 分钟）
   □ 阅读章摘要
   □ 回顾人物档案
   □ 确认时间线位置
   □ 检查待回收伏笔

2. 写作中（持续）
   □ 每 500 字：快速自检
   □ 场景转换：标记检查
   □ 新设定引入：确认约束
   □ 避免流水账（详略得当）

3. 写作后（20 分钟）
   □ AI 自动审查（一致性 + 流水账检测）
   □ 文笔质量评估
   □ 人工审查
   □ 修正问题
   □ 更新摘要
```

### 紧急检查清单（3 分钟）
```
□ 人物名字/特征是否一致
□ 时间顺序是否合理
□ 场景位置是否清晰
□ 是否有明显逻辑错误
□ 是否违反核心设定
□ 是否有流水账倾向
□ 文笔质量是否合格
```

### 常见错误速查
| 错误类型 | 警示词 | 检查方法 |
|----------|--------|----------|
| 时间混乱 | "突然"、"第二天" | 画时间线 |
| 人物 OOC | "居然"、"没想到" | 对比档案 |
| 设定冲突 | "记得之前" | 检索前文 |
| 逻辑断裂 | "就这样" | 因果链检查 |
| **流水账** | "然后"、"接着"、"很" | 详略检查 + 展示检测 |
| **文笔差** | 陈词滥调、无细节 | 五感检查 + 比喻评估 |
