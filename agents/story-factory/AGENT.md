## 📚 番茄风格核心规则

### 黄金三章

| 章节 | 任务 | 必含元素 |
|------|------|----------|
| 第 1 章 | 金手指觉醒 | 压抑→觉醒→希望 |
| 第 2 章 | 小冲突 | 展示能力 |
| 第 3 章 | 首次打脸 | 爽点释放 |

### 爽点密度

| 周期 | 爽点类型 | 数量 |
|------|----------|------|
| 每章 | 小爽点 | 1-2 个 |
| 每 3 章 | 中爽点 | 1 个 |
| 每 10 章 | 大爽点 | 1 个 |
| 每卷 | 超大爽点 | 1 个 (卷终高潮) |

### 质检标准

| 指标 | 标准值 |
|------|--------|
| 爽点密度 | >1.0 个/千字 |
| 对话比例 | 20-40% |
| 字数/章 | 2000-3500 字 |
| 章末钩子 | 必须有 |

## 📝 创作流程

```
1. 读取大纲 → 2. 创作正文 → 3. 自检 → 4. 更新记忆 → 5. 提交
```

**详细步骤**:

1. **读取大纲**
   - 查看主大纲当前进度
   - 确认本章核心事件
   - 确认伏笔埋设

2. **创作正文**
   - 按番茄风格创作
   - 保证爽点密度
   - 设计章末钩子

3. **自检**
   ```bash
   story check <chapter>
   ```
   - 爽点密度 >1.0
   - 对话比例 20-40%
   - 字数 2000-3500
   - 章末有钩子

4. **更新记忆**
   ```bash
   story memory update
   ```
   - 更新每日记忆
   - 更新项目记忆 (每 3 章)
   - 检查伏笔超期

5. **提交**
   ```bash
   git add -A && git commit -m "完成第 XXX 章" && git push
   ```

## 📊 写作技巧

### 爽点类型

| 类型 | 公式 | 关键词 |
|------|------|--------|
| 打脸 | 被看不起→隐忍→反击→震惊 | 嘲讽/震惊/跪下/后悔 |
| 收获 | 努力→获得→展示→规划 | 获得/奖励/突破/系统 |
| 成长 | 瓶颈→突破→验证 | 领悟/突破/晋升/终于 |
| 守护 | 重要的人被欺→主角赶到→护短 | 动我/保护/谁敢 |
| 复仇 | 仇恨→隐忍→清算 | 报仇/复仇/血债血偿 |
| 装逼 | 隐藏→被质疑→展示→震惊 | 淡淡/从容/不过如此 |
| 种田 | 从零开始→建设→成效 | 建设/发展/壮大 |
| 感情 | 相遇→互动→暧昧→确认 | 心动/脸红/表白 |

### 钩子类型

| 类型 | 示例 | 强度 |
|------|------|------|
| 新危机 | 就在这时，门外传来脚步声 | 强 |
| 新发现 | 他打开盒子，瞳孔骤缩 | 强 |
| 新人物 | 一个从未见过的男人出现 | 中 |
| 反转 | 她笑了：你以为我只有这张底牌 | 强 |
| 倒计时 | 距离爆炸还有 3 分钟 | 极强 |
| 疑问 | 这背后，到底隐藏着什么 | 中 |
| 情绪 | 他不知道，这一别竟是永别 | 中 |
| 选择 | 摆在他面前的，只有两条路 | 中 |
| 威胁 | 三日后，我会再来 | 强 |
| 悬念 | 那个神秘人，究竟是谁 | 中 |
