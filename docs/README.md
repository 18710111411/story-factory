## 📋 文档分类

### 01_项目文档
- **项目概览**: 项目背景、目标、范围
- **技术规范**: 技术标准、代码规范、质量要求
- **进度报告**: 周报、月报、里程碑报告

### 02_合同文档
- **合同模板**: 各类合同模板（AI 制作、服务、保密等）
- **已签署合同**: 已签署合同扫描件/电子版

### 03_会议记录
- **内部会议**: 团队会议、技术讨论
- **客户会议**: 需求沟通、项目评审
- **模板**: 会议记录标准格式

### 04_工作流程
- **推送流程**: GitHub 推送规范
- **审核流程**: 内容审核、合同审核
- **协作规范**: 团队协作规则

### 05_知识库
- **FAQ**: 常见问题解答
- **最佳实践**: 工作经验总结
- **参考资料**: 外部资源链接

## 📝 文档规范

### 命名规范
- 文件名：`YYYY-MM-DD_主题.md`
- 文件夹：`序号_分类名称/`
- 示例：`2026-04-05_喜磨坊项目启动会.md`

### 内容结构
```markdown
# 标题

## 基本信息
- 日期：YYYY-MM-DD
- 参与人：xxx
- 主题：xxx

## 内容

## 结论/待办

## 附件
```

### 版本控制
- 草稿：`v0.1`, `v0.2`...
- 正式版：`v1.0`, `v2.0`...
- 修订版：`v1.1`, `v1.2`...

## 🚀 快速开始

### 新建文档
```bash
# 1. 创建文件
touch docs/01_项目文档/2026-04-05_新文档.md

# 2. 编辑内容
# 使用模板（见下方）

# 3. 提交推送

git push origin master
```
